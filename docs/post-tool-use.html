[Post Tool Use]

scut claude hook post-tool-use and scut codex hook post-tool-use format files after successful edit tool calls. Both paths apply gofmt for Go source and goldmark-prettier-markdown for Markdown, respect root .prettierignore and .scutignore files, and report changed files back to the agent via additionalContext.

// 01Overview

Fires after an agent edit tool succeeds. The Claude command handles Write/Edit style inputs with a single file_path. The Codex command handles direct file_path inputs and apply_patch payloads that may touch multiple files. Formatting happens in-place, skips ignored paths, and injects a short additionalContext message when it changes files so the agent knows disk contents may differ from the original tool input.

// 02Input

The Claude command deserializes stdin as claudecode.PostToolUseInput. The Codex command deserializes stdin as codex.PostToolUseInput. The core tool fields are intentionally parallel:

FieldTypeDescription
(base fields) claudecode.Input / codex.TurnInput Common session fields. Codex also carries turn_id; Claude carries permission_mode, agent_id, and agent_type.
tool_name string Name of the tool that was called (e.g., Bash, Edit, mcp__github__search_repositories)
tool_use_id string Unique identifier for this tool invocation
tool_input json.RawMessage The input that was passed to the tool — shape varies per tool
tool_response json.RawMessage The tool's response — shape varies per tool

Path extraction

claudecode.PostToolUseInput.FilePath() extracts tool_input.file_path from the raw JSON and returns "" when absent, empty, or non-string. codex.PostToolUseInput.FilePaths() extracts a direct file_path and also parses apply_patch text from command, patch, or input. For patches, added, updated, and moved files are returned; deleted files are skipped. Relative Codex patch paths are resolved against cwd.

// 03Output

Written to stdout as claudecode.PostToolUseOutput or codex.PostToolUseOutput.

FieldTypeDescription
(base fields) BaseOutput continue, stopReason, suppressOutput, systemMessage
decision *Decision Set to "block" to reject the tool result
reason *string Explanation sent to Claude when blocking
additionalContext *string Legacy top-level context field; formatter notifications use hookSpecificOutput.additionalContext
updatedMCPToolOutput json.RawMessage Replacement output for MCP tools — Claude sees this instead of the original response
hookSpecificOutput event-specific hook output Nested event-specific output with hookEventName and additionalContext. Claude also supports optional updatedToolOutput.

When formatting changes one or more files, the implementation returns hookSpecificOutput.additionalContext with hookEventName set to PostToolUse. Skips and unchanged files still return an empty PostToolUseOutput{}.

// 04Dispatch Flow

  1. Decode the agent-specific PostToolUseInput from stdin
  2. Extract candidate paths: Claude uses in.FilePath(); Codex uses in.FilePaths()
  3. For each path, fs.Stat(fp) — skip if the file does not exist
  4. Check root .prettierignore and .scutignore files — skip if the path is ignored
  5. Switch on filepath.Ext(fp) to select a formatter — skip if no formatter matches
  6. afero.ReadFile(fs, fp) — skip on read error
  7. Call formatter — skip if result is nil (declined) or unchanged
  8. afero.WriteFile(fs, fp, formatted, info.Mode()) — preserves original file permissions
  9. Write PostToolUseOutput{} to stdout, including nested additionalContext only when the hook changed at least one file
noteEvery bail path writes an empty output and returns nil. The hook never errors on formatting failures — it silently skips.

// 05Byte Formatters

Formatters live in the internal/format package and are pure functions with signature func(src []byte) ([]byte, error):

FunctionExtensionsBackendBehavior on bad input
format.FormatGo .go go/format.Source Returns nil, nil (syntax error — let the compiler catch it)
format.FormatMarkdown .md, .mdx goldmark + goldmark-prettier-markdown with ProseWrapPreserve. Parser extensions enabled for GFM (tables, strikethrough, task checkboxes), footnotes, and definition lists — renderer-only, no HTML renderers added. Returns nil, nil (parse error)

Returning nil, nil means "decline to format" — the command skips the write.

// 06Ignore Files

Ignore handling lives in internal/formatignore. The hook walks upward from the target file directory and uses the first directory containing .git, .prettierignore, or .scutignore as the formatter root. It then loads root .prettierignore followed by root .scutignore; both files use gitignore-style patterns through go-git's plumbing/format/gitignore matcher, and later patterns win.

note.scutignore is loaded after .prettierignore, so it can add scut-specific exclusions or re-include paths with ! patterns.

// 07File I/O via afero.Fs

The command receives afero.Fs via Kong dependency injection (see kong-base-setup). In production this is afero.OsFs (real filesystem). In tests it's afero.MemMapFs (in-memory), which allows seeding files and verifying writes without touching disk.

// 08Adding a New Formatter

  1. Add a func Format<Name>(src []byte) ([]byte, error) function to internal/format/format.go (or a sibling file in the same package)
  2. Add test cases under internal/format/ — pure bytes in, bytes out
  3. Add a case to the extension switch in both internal/cmd/claude/hook/posttooluse.go and internal/cmd/codex/hook/posttooluse.go that selects format.Format<Name>
  4. Add dispatch test cases in internal/cmd/claude/hook/posttooluse_test.go and internal/cmd/codex/hook/hook_test.go
