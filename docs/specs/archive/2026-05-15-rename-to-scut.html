// 01Purpose

doneThis rename was executed and verified on 2026-05-15. All 17 plan tasks completed: the Go module path, binary, GitHub repo (ajbeck/botctrl → ajbeck/scut, permanent redirect in place), local working directory, log directory, and Claude Code memory directory were migrated; two commits landed on getting-started; tests green; bin/scut builds. This spec is retained as a historical record and archived under docs/specs/archive/. The PR/merge of getting-started → main remains a separate decision tracked outside this spec.

The product is being renamed from botctrl to scut. The new name is a deliberate triple-reference: in the Bobiverse universe (Dennis E. Taylor) SCUT is the Subspace Communications Universal Transceiver — the FTL protocol layer that links every Bob across light-years; in everyday English scut work is the unglamorous grinding labor an assistant does on behalf of someone else; and the four-letter lowercase token reads as a Unix tool of the same family as grep, awk, sed. All three meanings describe what this product is: the protocol-layer assistant that does the grunt work on behalf of LLM agents.

This spec covers the mechanics of the move. There is no scope creep — no API changes, no new features, no behavior changes. The diff is large but textual.

noteThis spec assumes the just-completed internal/cmd/claude/config/ package work (see 2026-05-14-config-command) is currently uncommitted on getting-started. Phase 0 commits that work under the old name before the rename starts. If state has drifted, re-read Phase 0 carefully.

// 05Phase 1 — Local rename

All textual edits happen on the still-named local directory. Nothing on disk moves yet, nothing on GitHub changes yet. This is the largest diff but the most reversible state: a single git reset --hard HEAD backs everything out.

Step 1 — Bulk replacement of unambiguous tokens

The module path is the most precise token in the codebase. Replace it first because it has zero false-positive risk:

module-path-replace.sh SHELL 
# Replace the full module path in every text file
grep -rl 'github.com/ajbeck/botctrl' --include='*.go' --include='*.mod' --include='*.sum' --include='*.html' --include='*.md' . \
  | xargs sed -i '' 's|github.com/ajbeck/botctrl|github.com/ajbeck/scut|g'

Step 2 — Rename the cmd directory

cmd-rename.sh SHELL 
git mv cmd/botctrl cmd/scut

Step 3 — Replace remaining standalone botctrl tokens

Now do a broader replacement for the bare token. This is less surgical — some matches will be inside prose, code blocks, or filenames. Mitigation: review each match before committing the replacement.

survey-remaining.sh SHELL 
# Survey what's left first — do not blindly sed
grep -rn 'botctrl' . --exclude-dir=.git --exclude-dir=bin

Categories of matches to expect, and how to treat each:

CategoryExampleAction
CLI invocation in prose / code blocksbotctrl claude config installReplace with scut claude config install.
owns() prefix string literals"botctrl", "botctrl ", "botctrl\t"Replace tokens with scut equivalents.
Test fixture command strings"botctrl claude hook post-tool-use"Replace with scut equivalent.
Log dir paths~/.botctrl/logging/Replace with ~/.scut/logging/.

Step 4 — Verify before committing

  1. mage fmt — should produce no diff if Step 1 / Step 3 left files in shape.
  2. mage vet — clean.
  3. mage test — clean, including the config-command tests whose fixtures were updated.
  4. mage build — produces bin/scut.
  5. Re-run the grep from Step 3: grep -rn 'botctrl' . --exclude-dir=.git --exclude-dir=bin. Expected output: only the asset filenames scut-docs.css / scut-docs.js (intentionally preserved) and the rename spec's prose discussing the rename.
haltIf mage test fails at this point, STOP. Do not commit. The most likely cause is a test fixture string that was missed by the bulk replace. Diagnose, fix, retry.

// 07Phase 3 — Coordinated GitHub + filesystem rename

The irreversible-ish bit. Each step is small and verifiable. Order matters.

session handoffStep 5 renames the local working directory. Any Claude Code session whose tool-call cwd points at the old path becomes useless after that point — every Bash tool call fails with "directory does not exist." Plan: complete Phase 3 Steps 1–4 in the current session, close it, run Steps 5–6 from a fresh terminal, then open a new Claude Code session in the renamed directory to continue with Step 7+. The Step 4 memory backup is what makes the handoff safe.

  1. STOP & CONFIRM — push the two commits to origin/getting-started under the still-botctrl name. The push works fine even though the code now references github.com/ajbeck/scut; Git does not validate module-path-to-remote-URL coherence. After push: gh pr list / gh repo view should still show the botctrl name.

  2. STOP & CONFIRM — rename the GitHub repository:

    gh-rename.sh SHELL 
    gh repo rename scut --repo ajbeck/botctrl --yes

    GitHub maintains a permanent HTTP redirect from github.com/ajbeck/botctrl to github.com/ajbeck/scut. Existing clones, links, and bookmarks continue to work.

  3. Update the local remote URL so future git push / git fetch go to the new URL directly rather than redirecting:

    remote-rename.sh SHELL 
    git remote set-url origin git@github.com:ajbeck/scut.git
git remote -v       # verify both fetch and push lines
git fetch origin    # verify connectivity

  4. Back up the Claude Code memory directory before moving it. The memory dir holds MEMORY.md plus the feedback memories built up across sessions. If anything goes wrong with the mv in the next step, the backup is the recovery path.

    memory-backup.sh SHELL 
    cp -r /Users/aj/.claude/projects/-Users-aj-Developer-repos-ajbeck-botctrl \
      /tmp/claude-memory-botctrl-backup-$(date +%Y%m%d-%H%M)

  5. Rename the local working directory:

    dir-rename.sh SHELL 
    mv ~/Developer/repos/ajbeck/botctrl ~/Developer/repos/ajbeck/scut

  6. Migrate the Claude Code memory dir to its new path-encoded name:

    memory-migrate.sh SHELL 
    mv /Users/aj/.claude/projects/-Users-aj-Developer-repos-ajbeck-botctrl \
   /Users/aj/.claude/projects/-Users-aj-Developer-repos-ajbeck-scut

  7. cd into the new path. Verify everything still works:

    final-verify.sh SHELL 
    cd ~/Developer/repos/ajbeck/scut
git status               # clean
git remote -v            # points to ajbeck/scut
mage fmt && mage vet && mage test && mage build
ls bin/scut              # new binary name

  8. Confirm Claude Code memory survived the migration by starting a new session in the renamed directory and verifying it sees the existing MEMORY.md entries. Once confirmed, the /tmp/ backup from Step 4 can be deleted at leisure.

// 09Risks & Mitigations

RiskLikelihoodMitigation
Bulk sed over-matches and replaces botctrl inside a context where it should be preserved (e.g., asset filenames, historical rename references) medium Phase 1 splits the replacement into two passes: Step 1 replaces only the unambiguous full module path; Step 3 surveys remaining matches before changing them. The CSS/JS filenames and the rename references are explicitly preserved per §03.
Test fixture string mismatch missed by the replace, surfaced as a runtime test failure low Phase 1 Step 4 runs mage test before commit. A failing test halts Phase 1; Phase 2 does not start.
Claude Code memory dir migration fails or corrupts during mv low Phase 3 Step 4 copies the memory dir to /tmp/ before moving the live dir. Recovery: cp -r from the backup to the new path-encoded location.
Stale Git remotes on other machines medium Other clones will continue to work via GitHub's permanent redirect. Update them lazily — next time you cd into one of them, run git remote set-url origin.
A push to origin/getting-started happens between Phase 2 and Phase 3, e.g. from an editor's auto-sync low Push still works under either repo name thanks to the redirect. No corrective action required even if it happens.
Existing ~/.botctrl/ log dir on the user's machine becomes orphaned n/a Verified absent — the user has never run with logging enabled. No migration code or backward-compat path needed.
The getting-started branch name becomes stale after the rename cosmetic Out of scope. Branch is short-lived and gets merged to main soon — separate decision.
