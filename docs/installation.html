[install]

scut ships as GitHub Release tarballs for macOS and Linux. The install script selects the current platform, downloads the matching artifact, verifies it against checksums.txt, and places the binary on the user's PATH.

// 01Overview

The primary install path is release-based. Released binaries have their version injected from the immutable vM.m.p git tag. Source installs use Go build information when available, so go install github.com/ajbeck/scut@vM.m.p reports that module version. Local builds keep the default development version unless a build sets linker flags through Mage.

noteThe release workflow also deploys the documentation site from the same main commit used for the release, so the hosted install instructions match the published artifacts.

// 02Install Script

install.sh 
curl -fsSL https://raw.githubusercontent.com/ajbeck/scut/main/install.sh | sh

The default destination is ~/.local/bin/scut. Pass --version to pin a release and --bin-dir to choose another install directory:

note~/.local/bin is the per-user executable location from the XDG Base Directory convention. Use --bin-dir /usr/local/bin when you intentionally want a system-wide install managed by an administrator.
pinned install 
curl -fsSL https://raw.githubusercontent.com/ajbeck/scut/main/install.sh | sh -s -- --version v0.1.0 --bin-dir /usr/local/bin
FlagBehavior
--version VERSION Install a specific GitHub Release. The leading v is optional.
--bin-dir DIR Install scut into the given directory instead of ~/.local/bin.

// 03Release Assets

Each stable release publishes one tarball per platform plus a checksum manifest:

release assets 
scut-v0.1.0-darwin-amd64.tar.gz
scut-v0.1.0-darwin-arm64.tar.gz
scut-v0.1.0-linux-amd64.tar.gz
scut-v0.1.0-linux-arm64.tar.gz
checksums.txt

The tarballs contain a single executable named scut. The install script chooses the artifact from uname -s and uname -m, downloads checksums.txt, and refuses to install when the SHA-256 digest does not match.

// 04Source Installs

Go users can still install from source:

go install 
go install github.com/ajbeck/scut@latest

For local development, use Mage so the JSON v2 experiment and build metadata are set consistently:

local build 
mage build

// 05Release Flow

  1. Code lands on main.
  2. The Release workflow is manually dispatched with a stable semver version.
  3. The workflow checks out main, creates immutable vM.m.p, and updates movable vM.m and vM tags.
  4. The reusable build workflow builds all platform artifacts with RELEASE_VERSION=vM.m.p.
  5. The release job creates the GitHub Release and uploads tarballs plus checksums.txt.
  6. The docs job deploys docs/ from the same release commit to GitHub Pages.
