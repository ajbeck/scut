[Codex Config]

scut codex config manages Codex hooks.json files. It writes command-hook entries that invoke scut codex hook ..., preserves unrelated hook groups and top-level keys, and gives users install, uninstall, and status operations that match the Claude config command shape where the Codex model overlaps.

// 01Overview

Codex can load hooks from hooks.json or inline [hooks] tables in config.toml. scut writes hooks.json only. That avoids TOML rewrites, keeps ownership narrow, and follows the Codex guidance to prefer one hook representation per config layer because Codex merges both forms and warns when both exist.

notescut does not write [features].hooks = true. Codex documents hooks as enabled by default; codex_hooks is only a deprecated alias.

// 02Commands

codex-config.sh SH 
# Project scope: .codex/hooks.json
scut codex config install

# User scope: ~/.codex/hooks.json
scut codex config install --scope=user

# Preview without writing
scut codex config install --dry-run

# Install an explicit event set
scut codex config install --only=session-start,post-tool-use

# Remove all scut-owned Codex hook groups
scut codex config uninstall

# Inspect both scopes
scut codex config status --scope=both --json
OperationBehavior
install Without --only, installs only post-tool-use, the Codex hook with real formatting behavior. Explicit --only values can install any known Codex hook event.
uninstall Without --only, removes all scut-owned Codex hook groups. With --only, removes only those event slugs.
status Lists scut-owned command hooks in project, user, or both scopes. --json emits structured output.

// 03Model

The file model mirrors Codex's documented hook shape: top-level hooks, event keys such as PostToolUse, matcher groups, then command handlers. Foreign top-level keys are stored in an inline fallback map and round-tripped through deterministic JSON output.

hooks.json JSON 
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "apply_patch|Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "scut codex hook post-tool-use",
            "statusMessage": "Formatting..."
          }
        ]
      }
    ]
  }
}

// 04Ownership

scut owns only command hooks whose command starts with scut codex hook , scut codex --log hook , or a scut codex --log-level=... hook ... invocation. Uninstall and status ignore foreign hook groups, including other commands inside the same event. Unknown fields on existing hook groups and hook entries are round-tripped through inline fallback maps.
