[scut init]

scut init is the user-facing setup entrypoint. It detects supported coding agents, then delegates to the same Claude and Codex config installation functions used by the agent-specific command groups.

// 01Overview

The command does not maintain a second configuration model. Claude setup reuses scut claude config install internals, and Codex setup reuses scut codex config install internals. That keeps merge behavior, ownership rules, and dry-run output aligned with the low-level commands.

init.sh SH 
# Set up detected agents in project scope
scut init

# Force all supported agents
scut init --all

# Select one agent explicitly
scut init --codex

# Preview generated config files
scut init --all --dry-run

# Print the full rendered config during dry-run
scut init --all --dry-run --verbose

# Bake hook logging into generated commands
scut init --all --bake-log-level=debug

// 02Selection

ModeBehavior
default Installs for detected agents only. Project scope detection requires a local .claude/ or .codex/ directory. User scope detection checks the user config directory and then falls back to PATH lookup for the agent binary.
--claude / --codex Installs only the selected agent or agents, without detection gating.
--all Installs all supported agents, currently Claude Code and Codex.
--scope Selects project or user scope and is passed through to each agent-specific installer.
--bake-log / --bake-log-level Bakes scut logging flags into every generated hook command. --bake-log-level implies logging and omits the redundant bare --log flag.
--verbose With --dry-run, prints the full rendered config for each selected agent. Without it, dry-run prints a compact target summary.

// 03Output

Normal runs preflight every selected target before writing any file, then print one result line per installed agent plus a suggested scut doctor --scope=<scope> follow-up. Dry runs print a compact target summary by default; add --verbose to print the generated JSON that would be written by each agent's config installer. If no supported agents are detected in default mode, the command reports each skipped agent and exits successfully without writing files.
