[Gotools Doc Parity Plan]

Tighten scut gotools doc around go doc compatibility for human-readable output. The target is official lookup and flag parity for packages, symbols, methods, fields, current-package lookups, local package suffix shorthand, command packages, symbol formatting, resolver diagnostics, local replacement modules, and simple proxy fallback. It does not add JSON output and does not include GitHub Actions workflow work.

// 01Goal

Make scut gotools doc behave like the official Go 1.26 go doc command for the default text workflow agents and humans already expect.

target-output.txt ASCII 
$ scut gotools doc github.com/spf13/afero.Symlinker
package afero // import "github.com/spf13/afero"

type Symlinker interface {
        Lstater
        Linker
        LinkReader
}
    Symlinker is an optional interface in Afero. It is only implemented by the
    filesystems saying so.

// 02Decisions

DecisionChoiceReason
Default output Match go doc Agents and humans already understand the standard text format, and examples can be compared directly against Go's tool.
User model No relearning For Go-tool-inspired functionality, consumers should be able to use the out-of-the-box Go command mental model. Deviations must be explicit and justified.
Official reference Go 1.26.3 cmd/go/internal/doc Use the installed Go command source as the behavioral reference, especially parseArgs, parseSymbol, symbolDoc, printMethodDoc, printFieldDoc, and match.
Go rules reference /Users/aj/home/Developer/go-rules Follow the local Go 1.26 rules: standard library first, clear small helpers, structured errors where callers branch, behavior tests over interaction tests, and no new dependencies.
Lookup scope Full go doc lookup forms Support packages, top-level symbols, type methods, interface methods, struct fields, current-package lookup, and two-argument <pkg> <sym>[.<methodOrField>].
Lookup model Optional symbol object Represent package docs with Symbol == nil, symbol docs with a non-nil symbol and nil member, and method/field docs with a non-nil member. Keep required package and diagnostic paths as value strings so nil handling is limited to the real semantic branch.
Package suffix shorthand Local and known packages only Support shorthand such as json.decode, rand.Float64, template.new, and suffix paths such as ivy/value by searching stdlib, current module, and module cache indexes. Do not search or fetch remote modules from shorthand alone.
Package index tiers First matching local tier Return matches from the first non-empty tier: stdlib, then current module, then module cache. This keeps common shorthand such as json.decode interactive and avoids walking large module caches when a higher-priority local tier already contains matching packages.
Current package Match go doc No args should document the current package. Capitalized one-argument lookups such as Foo and Foo.Bar should resolve in the current package when the working directory is a buildable Go package.
Argument count Zero, one, or two Match official go doc: no args selects the current package, one arg is a Go-syntax-like lookup, and two args are <pkg> <sym>[.<methodOrField>]. More than two args is a usage error.
Package patterns Do not implement go list patterns Match official go doc: package path elements such as . and ... are not general package-list patterns. Local ./ and ../ directory lookups remain supported.
Command packages Add --cmd For package main, hide exported symbol summaries by default and show them when --cmd is set, matching official go doc.
Flag parity Design for all doc flags Existing --all, --short, --src, -u, and -c should be judged against official output. Add --cmd in the same design, even if execution is phased.
-C dir Defer Official go doc supports changing directories before lookup. Defer it because scut's injectable/in-memory filesystem model makes process chdir semantics more complicated than normal CLI state.
-http Defer Official go doc can serve browser docs through pkgsite. This plan targets deterministic terminal text for agents, not an interactive local HTTP server.
Multi-match symbols Match go doc When case-insensitive matching finds multiple symbols, print all matches rather than picking a winner. -c narrows matching to exact case.
Type-associated items Match go doc Type symbol output should include associated constructors, methods, typed constants, and typed variables in the same places official go doc does.
Machine output No JSON in this pass There is no concrete structured-data consumer yet. Avoid schema design until a real caller needs it.
Workflow split No GitHub Actions work Pull request and release workflows are a separate subsystem and should get their own plan.
Resolver parity Narrow, tested additions Add local replace support and simple proxy fallback only. Defer checksum DB, full GOVCS, and non-Git VCS.

// 03Architecture

Keep the command layer thin. internal/cmd/gotools should map Kong flags into godoc.Options and write one trailing newline. Official-style lookup belongs in internal/godoc, because the correct behavior depends on trying package candidates, parsing symbol[.methodOrField], checking whether formatting finds a symbol, and preserving enough attempted-package context for useful errors.

research findingThe official Go 1.26.3 command does not split package.Symbol once. Its parseArgs first tries the whole argument as a package, then tries dotted package prefixes after the final slash. Its formatter separately handles top-level symbols, methods, interface methods, and struct fields. scut should mirror that model instead of accumulating command-layer special cases.

// 05Task 1 - Lookup Engine

Replace command-layer string splitting with an official-style lookup engine in internal/godoc. The command should pass raw lookup arguments; godoc should decide whether the request means a package, top-level symbol, type member, current-package symbol, or package suffix shorthand.

The implementation should make Client.Doc resolve a request before parsing and formatting. Keep fetchers package-oriented; the lookup layer owns argument interpretation, package candidate order, current-package fallback, and the decision to try the next package when formatting does not find a symbol.

// 06Task 2 - Package Index

Add a local package index for official suffix shorthand without introducing remote search. Shorthand such as json.decode, rand.Float64, template.new, and suffix paths such as ivy/value should search only packages that are knowable locally.

The index is deliberately a discovery helper, not a fetcher. It should return full import paths for locally knowable packages, and Resolver.Fetch should still be responsible for loading source for the chosen full path.

Implementation note

Task 4 landed with formatter-owned visibility filtering. The client now parses packages with doc.AllDecls for every doc request, then the formatter hides unexported declarations, methods, and fields unless -u is set. This matches the upstream cmd/go/internal/doc shape and lets default type output render // Has unexported fields. instead of leaking go/doc's lower-level filtered marker. Source mode also suppresses the formatter's wrapped documentation after printing the source declaration, because official go doc -src relies on source comments already present in the declaration.

// 09Task 5 - Error Quality

Replace user-facing source fetcher not applicable failures with errors that name the package or symbol lookup and preserve useful resolver causes for debugging.

Implementation note

Task 5 introduced typed package and lookup errors in internal/godoc. Source fetchers still use ErrSourceNotApplicable internally to decline a route, but Resolver.Fetch converts an all-fetchers-declined result into PackageNotFoundError. The lookup layer continues across package candidates and suffix matches, then reports either a package miss for package-only lookups or a symbol/member miss that includes the attempted package list. This keeps CLI output clear while preserving typed errors for callers and tests.

Implementation note

Task 6 resolved local replacements through a dedicated ReplaceSourceFetcher. Current module parsing now records only local replacement directives with no replacement version, resolves relative paths against the module root, and ignores remote replacement targets. The fetcher uses longest module-prefix matching, reads the replacement checkout directly, and rejects package paths that would escape the replacement root before falling through to module cache and network fetchers.
